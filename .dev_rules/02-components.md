# Компоненты и UI‑библиотеки (shadcn/ui + Radix UI)

## 1. Работа с shadcn/ui
- **Никогда не редактируйте файлы в `components/ui/` после `npx shadcn add`** — при обновлении они будут перезаписаны.
- Для кастомизации создавайте копию в `components/custom-ui/` или оборачивайте компонент.
- Добавляйте компоненты только через CLI:
  ```bash
  npx shadcn@latest add button dialog sheet dropdown-menu
  ```
- При инициализации выбирайте **Radix UI** (рекомендуется) или Base UI.

## 2. Прямая работа с Radix UI
- В 90% случаев достаточно shadcn-обёрток. Обращайтесь к Radix напрямую, если:
  - нужен очень кастомный компонент, отсутствующий в shadcn;
  - требуется минимизировать бандл (использовать только примитивы);
  - вы создаёте компонент с нуля, взяв за основу Radix.
- Пример прямого импорта:
  ```tsx
  import * as SelectPrimitive from "@radix-ui/react-select"
  ```

## 3. Доступность (a11y) — обязательный минимум
- shadcn/ui уже обеспечивает базовую доступность (роли, фокус, клавиатура). Не ломайте её.
- **Обязательно**:
  - `aria-label` / `aria-labelledby` на иконках‑кнопках.
  - `role="status"` для живых областей (тосты, загрузка).
  - Контраст цветов проверяйте (OKLCH упрощает подбор).
  - Используйте `asChild` в Radix-компонентах для семантической вёрстки.
  - Добавляйте `skip to content` ссылку для навигации с клавиатуры.

## 4. Динамические импорты (code splitting)
- Тяжёлые компоненты, редакторы, графики загружайте динамически:
  ```tsx
  const HeavyChart = dynamic(() => import("@/components/HeavyChart"), {
    ssr: false,
    loading: () => <Skeleton className="h-96 w-full" />
  })
  ```
- Для библиотек, которые не работают на сервере, используйте `ssr: false`.

## 5. Иконки
- Предпочитайте `lucide-react` (уже идёт с shadcn). 
- Иконки должны быть обёрнуты в семантический компонент с `aria-hidden` (если декоративные) или `aria-label` (если интерактивные).

## 6. Работа с изображениями
- Всегда используйте `next/image` (даже для локальных картинок).
- Настройте `remotePatterns` в `next.config.js` для внешних изображений.
- Используйте атрибуты `priority` для LCP-изображений, `placeholder="blur"` для локальных.

## 7. Формы (кратко)
- Комбинация: `react-hook-form` + `zod` + Server Action.
- Типизируйте данные с помощью `zod.infer`.
- Используйте `useActionState` (React 19) для управления состоянием отправки и ошибками.
```
